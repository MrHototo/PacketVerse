Filter syntax

Filters are evaluated per-packet, like Wireshark's display filter — matching traffic is shown everywhere (3D view, packet list, dashboard) and everything else is hidden.

  • dns, tls, arp — bare protocol / tag match
  • ip.addr==10.0.0.5 · ip.src== / ip.dst==
  • eth.src== / eth.dst== — MAC address
  • tcp.port==443, udp.port==53 (protocol-scoped)
  • tcp.flags.syn==1, tcp.flags.ack==1, tcp.flags.rst==1
  • dns.qry.name==example.com, dns.qry.type==AAAA
  • tls.handshake.extensions_server_name ~ /example/, tls.record.version
  • http.request.method==GET
  • vlan.id==10, icmp.type==8
  • bytes>10000, packets>50, duration>5
  • ip.addr ~ /^10\./ — regex match
  • dns and udp.port==53, (tls or http) and bytes>1000
  • not arp, !broadcast
  • http.request.uri contains "login" — substring match
  • ip.addr matches "^10\\." — matches is an alias for ~
  • tcp.dstport in {80, 443, 8443} — set membership
  • tcp.analysis.retransmission, tcp.analysis.duplicate_ack, tcp.stream==2
  • tls.handshake.ciphersuite, tls.cert.subject, http.host, http.response.code==404

Plain text with no operators falls back to a simple substring search. Press Enter to snap the camera to the filtered results.
