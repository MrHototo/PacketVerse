Filter syntax

A real implementation of Wireshark's own display-filter grammar — not a fixed list of examples. Full reference: docs/FILTER_SYNTAX.md. Matching traffic is shown everywhere (3D/2D view, packet list, dashboard); everything else is hidden.

  • Existence: dns, tls, arp, tls.handshake, dns.cname — bare protocol/field presence
  • Comparison: == != > < >= <= or eq ne gt lt ge le; quantify repeating fields with any/all: all tcp.port > 1024
  • Search: http.request.uri contains "login" · dns.qry.name matches "^www\." (or ~) — regex, case-insensitive by default, use (?-i) for case-sensitive
  • Membership + ranges: tcp.port in {80, 443, 8443} · tcp.port in {443, 4430..4434} · ip.addr in {10.0.0.1..10.0.0.9}
  • CIDR: ip.addr == 10.0.0.0/24, ipv6.addr == 2001:db8::/32
  • Logic: and/&&, or/||, xor, not/!, parentheses — precedence: not > and > xor > or
  • Bitwise AND: tcp.flags & 0x02 — true when the SYN bit is set
  • Slices: eth.src[0:3] == 00:11:22, frame[-4:], field[1,3-5,9:]
  • Literals: hex 0x, binary 0b, octal 012, MAC/byte sequences aa:bb:cc, raw strings r"..."
  • Functions: upper() lower() len() count() vals() dec() hex() abs() max() min() ip_rfc1918() ip_multicast() ip_linklocal()
  • Examples: tls && tcp.port == 853 · tls.handshake.type == "client_hello" · dns.a == 93.184.216.34 · tcp.analysis.retransmission · upper(http.host) contains "GOOGLE"
  • Fields not yet deep-decoded gracefully fall back to a protocol-presence test instead of always failing. Not yet supported: macros ($name), field references, arithmetic (+ - * /), the layer (#) and raw (@) operators.

Plain text with no operators falls back to a simple substring search. Press Enter to snap the camera to the filtered results.
